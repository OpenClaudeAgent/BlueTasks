# Accessibilité (BlueTasks)

## Vérifications automatiques

- **ESLint** : le workspace `web/app` inclut `eslint-plugin-jsx-a11y` (règles *recommended* sur les fichiers `*.tsx`). Lancer :

  ```sh
  npm run lint --workspace web/app
  ```

  À la racine du monorepo : `npm run lint`.

## Tests manuels dans le navigateur

- **Extension [axe DevTools](https://www.deque.com/axe/devtools/)** (ou équivalent) : analyse rapide des pages après build ou en dev.
- **Navigation clavier** : `Tab` / `Shift+Tab`, activation des boutons avec `Entrée` ou `Espace`, fermeture des dialogues (Radix gère en partie `Escape` selon le composant).
- **Zoom** : vérifier la lisibilité à 200 % dans les paramètres du navigateur.

## Pistes d’amélioration continue

- Ajouter des tests automatisés (ex. Playwright + `@axe-core/playwright`) sur les flux critiques (liste de tâches, dialogue paramètres) si vous industrialisez la CI.
