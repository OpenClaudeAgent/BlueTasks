# Accessibility (BlueTasks)

## Automated checks

- **ESLint** — the `web/app` workspace enables `eslint-plugin-jsx-a11y` (recommended rules on `*.tsx`). Run:

  ```sh
  npm run lint --workspace web/app
  ```

  From the monorepo root: `npm run lint`.

## Manual checks in the browser

- **[axe DevTools](https://www.deque.com/axe/devtools/)** (or similar) — quick passes after `npm run build` or in dev.
- **Keyboard** — `Tab` / `Shift+Tab`, activate buttons with **Enter** or **Space**, dismiss dialogs with **Escape** where Radix provides it.
- **Zoom** — confirm readability at **200%** in the browser.

## Optional CI automation

For automated a11y regression in CI, **Playwright** can run **`@axe-core/playwright`** on critical flows (e.g. task list, Settings). This is optional; ESLint `jsx-a11y` remains the default gate.
