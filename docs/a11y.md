# Accessibility (BlueTasks)

## Automated checks

- **ESLint** — the `web/app` workspace enables `eslint-plugin-jsx-a11y` (recommended rules on `*.tsx`). Run:

  ```sh
  npm run lint --workspace web/app
  ```

  From the monorepo root: `npm run lint`.

- **axe-core (Playwright)** — [`e2e/a11y-axe.spec.ts`](../e2e/a11y-axe.spec.ts) (helper: [`e2e/a11y-axe-helpers.ts`](../e2e/a11y-axe-helpers.ts)) runs **`@axe-core/playwright`** on the **production-shaped** app (build + API, same as other e2e). **UI locale is forced to English** (`gotoWithEnglish` / `resetBoard`) so roles and `title` attributes match `locales/en.ts`. Scenarios exercised (each ends with a full-page axe scan):
  - Shell: **Today** (empty), **All / Upcoming / Anytime / Done** (empty), **unknown SPA path**
  - **Settings**: General; Categories (empty list); Categories (after adding one category)
  - **Task card**: expanded (Lexical + toolbar), collapsed; **two tasks** on Today
  - **Popovers**: **due date** (calendar open), **category** (with a seeded category), **time estimate**, **recurrence**, **timer edit**
  - **Lists**: **All** with one open + one done task; **Done** with one completed task

  CI runs these as part of the **E2E** jobs (`npm run test:scenario` / Playwright sharding). Quick local run:

  ```sh
  npx playwright test e2e/a11y-axe.spec.ts
  ```

  Root shortcut: `npm run test:a11y`.

  _Not covered by axe e2e today (still use jsx-a11y + manual passes): e.g. **sidebar resize** interaction, **priority** footer cycling, **Lexical** toolbar matrix (separate [`editor-toolbar.spec.ts`](../e2e/editor-toolbar.spec.ts)), **desktop Tauri** shell._

## Manual checks in the browser

- **[axe DevTools](https://www.deque.com/axe/devtools/)** (or similar) — quick passes after `npm run build` or in dev.
- **Keyboard** — `Tab` / `Shift+Tab`, activate buttons with **Enter** or **Space**, dismiss dialogs with **Escape** where Radix provides it.
- **Zoom** — confirm readability at **200%** in the browser.

## Policy

**ESLint `jsx-a11y`** catches many issues at authoring time; **axe** in e2e catches runtime/DOM issues (contrast, landmarks, labels) on real pages. Both are expected to stay green in CI.
